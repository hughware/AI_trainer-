### 完整可运行代码

```python
# 导入必要的库
import numpy as np
from PIL import Image
import onnxruntime as ort


# 定义预处理函数，用于将图片转换为模型所需的输入格式
def preprocess(image_path):
    input_shape = (1, 1, 64, 64)    # 模型输入期望的形状 (N, C, H, W)
    img = Image.open(image_path).convert('L')    # 打开图像并转换为灰度图
    img = img.resize((64, 64), Image.ANTIALIAS)    # 调整图像大小到64x64
    img_data = np.array(img, dtype=np.float32)    # 转为numpy数组
    img_data = np.expand_dims(img_data, axis=0)  # 添加 batch 维度
    img_data = np.expand_dims(img_data, axis=1)  # 添加 channel 维度
    assert img_data.shape == input_shape, f"Expected shape {input_shape}, but got {img_data.shape}"
    return img_data


# 定义情感类别与数字标签的映射表
emotion_table = {'neutral':0, 'happiness':1, 'surprise':2, 'sadness':3, 'anger':4, 'disgust':5, 'fear':6, 'contempt':7}

# 加载模型
ort_session = ort.InferenceSession('onnx/emotion-ferplus.onnx')

# 加载本地图片并进行预处理
input_data = preprocess('onnx/img_test.png')

# 准备输入数据
ort_inputs = {ort_session.get_inputs()[0].name: input_data}

# 运行模型，进行预测
ort_outs = ort_session.run(None, ort_inputs)

# 解码模型输出，找到预测概率最高的情感类别
predicted_label = np.argmax(ort_outs[0])

# 根据预测的标签找到对应的情感名称
predicted_emotion = list(emotion_table.keys())[predicted_label]

# 输出预测的情感
print(f"Predicted emotion: {predicted_emotion}")
```

## 代码解析

### 各空白处填写说明

1. **`emotion_table = {'neutral':0, 'happiness':1, 'surprise':2, 'sadness':3, 'anger':4, 'disgust':5, 'fear':6, 'contempt':7}`**
   - 定义8种情感类别与数字标签的映射关系
   - 题目中已明确给出映射表内容，直接填写即可

2. **`ort_session = ort.InferenceSession('onnx/emotion-ferplus.onnx')`**
   - 使用onnxruntime加载表情识别模型
   - 考试环境中模型路径为 D:\jupyter_notebook\onnx

3. **`input_data = preprocess('onnx/img_test.png')`**
   - 调用已定义的preprocess函数对测试图片进行预处理
   - 预处理包括：转灰度图、resize到64x64、转float32数组、添加batch和channel维度

4. **`ort_outs = ort_session.run(None, ort_inputs)`**
   - 第一个空填 `ort_session.run`
   - 第二个空填 `ort_inputs`
   - 执行模型推理，None表示获取所有输出

5. **`predicted_label = np.argmax(ort_outs[0])`**
   - np.argmax获取输出数组中最大值的索引
   - 该索引即为预测的情感类别编号

6. **`predicted_emotion = list(emotion_table.keys())[predicted_label]`**
   - 将emotion_table的键转为列表
   - 用预测的标签索引获取对应的情感名称字符串

### 图像预处理流程
- 转为灰度图（单通道，L模式）
- Resize到64x64（模型要求的输入尺寸）
- 转为float32类型的numpy数组
- 添加batch和channel维度，最终shape为(1, 1, 64, 64)

### 模型输入输出
- 输入shape：(1, 1, 64, 64) -- batch_size=1, channels=1(灰度), height=64, width=64
- 输出：8个类别的置信度分数，对应8种表情

## 交互流程设计（3.2.3.docx内容）

### 面部表情识别系统人机交互最优方案

#### 一、系统概述

本系统基于emotion-ferplus深度学习模型，通过ONNX格式部署，能够识别8种面部表情：中性(neutral)、快乐(happiness)、惊讶(surprise)、悲伤(sadness)、愤怒(anger)、厌恶(disgust)、恐惧(fear)、蔑视(contempt)。

#### 二、交互界面设计

界面采用上中下三栏布局：
- **上方区域**：图像输入区，支持摄像头实时采集或上传本地图片
- **中间区域**：面部检测预览区，显示检测到的人脸区域和表情标注
- **下方区域**：识别结果区，以雷达图展示8种表情的概率分布，突出显示置信度最高的表情

#### 三、最优交互流程

**方式一：静态图片识别**
1. 用户点击"上传图片"按钮，选择包含人脸的图片
2. 系统自动检测图片中的人脸区域，在预览区标注人脸框
3. 对检测到的人脸进行灰度转换和尺寸归一化（64x64）
4. 模型推理输出8种表情的概率分布
5. 在结果区以雷达图和文字形式展示识别结果

**方式二：实时视频识别**
1. 用户开启摄像头，系统实时采集视频帧
2. 每帧自动进行人脸检测和表情识别
3. 在视频画面上实时标注人脸框和表情标签
4. 侧边栏显示情绪变化趋势图

#### 四、优化策略

- **多人脸支持**：同时检测和识别画面中多个人脸的表情
- **情绪趋势分析**：记录一段时间内的表情变化，生成情绪波动曲线
- **预警机制**：当检测到异常表情（如持续悲伤、愤怒）时，系统给出预警提示
- **结果导出**：支持将识别结果和分析报告导出为PDF或CSV格式
