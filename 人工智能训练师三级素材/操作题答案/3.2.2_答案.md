### 完整可运行代码

```python
import onnxruntime
import numpy as np
from PIL import Image

# 加载ONNX模型
ort_session = onnxruntime.InferenceSession('onnx/mnist.onnx')

# 加载图像
image = Image.open('onnx/img_test.png').convert('L')  # 转为灰度图

# 图像预处理
image = image.resize((28, 28))  # 调整大小为MNIST模型的输入尺寸
image_array = np.array(image, dtype=np.float32)  # 转为numpy数组
image_array = np.expand_dims(image_array, axis=0)  # 添加batch维度
image_array = np.expand_dims(image_array, axis=0)  # 添加通道维度

# 返回模型输入列表
ort_inputs = {ort_session.get_inputs()[0].name: image_array}
# 执行预测
ort_outs = ort_session.run(None, ort_inputs)

# 获取预测结果
predicted_class = np.argmax(ort_outs[0])

# 输出预测结果
print(f"Predicted class: {predicted_class}")
```

## 代码解析

### 各空白处填写说明

1. **`ort_session = onnxruntime.InferenceSession('onnx/mnist.onnx')`**
   - 使用onnxruntime加载MNIST的ONNX模型
   - 考试环境中模型在 D:\jupyter_notebook\onnx 目录下

2. **`image = Image.open('onnx/img_test.png').convert('L')`**
   - 打开测试图片并转为灰度图（题目已给出'L'参数）
   - MNIST模型要求单通道灰度图输入

3. **`image = image.resize((28, 28))`**
   - MNIST模型标准输入尺寸为28x28像素

4. **`image_array = np.array(image, dtype=np.float32)`**
   - 将PIL图像转为numpy数组，数据类型为float32
   - 注意：此处第一个空填 `np.array`，第二个空填 `image`

5. **`image_array = np.expand_dims(image_array, axis=0)`**
   - 添加batch维度，shape从(28,28)变为(1,28,28)
   - 注意：此处第一个空填 `np.expand_dims`，第二个空填 `image_array`

6. **`image_array = np.expand_dims(image_array, axis=0)`**
   - 添加通道维度，shape从(1,28,28)变为(1,1,28,28)
   - 符合模型输入要求：(batch_size, channels, height, width)
   - 注意：此处第一个空填 `np.expand_dims`，第二个空填 `image_array`

7. **`ort_inputs = {ort_session.get_inputs()[0].name: image_array}`**
   - 此处空填 `ort_session.get_inputs`，获取模型输入节点信息

8. **`ort_outs = ort_session.run(None, ort_inputs)`**
   - 此处空填 `ort_session.run`，执行模型推理
   - None表示获取所有输出

9. **`predicted_class = np.argmax(ort_outs[0])`**
   - np.argmax获取输出数组中最大值的索引，即预测的数字类别(0-9)

### 图像预处理流程
- 转为灰度图（单通道）
- Resize到28x28（MNIST标准输入尺寸）
- 转为float32类型的numpy数组
- 添加batch维度和channel维度，最终shape为(1, 1, 28, 28)

## 交互流程设计（3.2.2.docx内容）

### 手写数字识别系统人机交互最优流程

#### 一、系统概述

本系统基于卷积神经网络（CNN）训练的MNIST模型，通过ONNX格式部署，实现对手写数字0-9的智能识别。系统提供画板手写输入和图片上传两种输入方式，满足不同场景需求。

#### 二、交互界面设计

界面采用简洁的上下布局：
- **上方区域**：手写画板区（支持鼠标/触屏手写输入）+ 图片预览区
- **中间区域**：操作按钮区（识别、清除画板、上传图片）
- **下方区域**：识别结果展示区（显示识别数字和各数字的概率分布柱状图）

#### 三、最优交互流程

**方式一：手写输入识别**
1. 用户在画板区域使用鼠标或触摸笔手写一个数字
2. 点击"识别"按钮，系统自动将画板内容转换为28x28灰度图像
3. 系统执行预处理和模型推理，在结果区展示识别的数字及置信度
4. 用户可点击"清除"按钮重新书写

**方式二：图片上传识别**
1. 用户点击"上传图片"按钮，选择本地手写数字图片
2. 系统在预览区显示图片缩略图
3. 自动进行灰度转换、尺寸调整等预处理
4. 展示识别结果

#### 四、优化策略

- **实时识别**：在手写输入过程中，当用户停笔超过1秒时自动触发识别，无需手动点击按钮
- **多数字识别**：支持在画板上书写多个数字，系统自动分割并逐个识别
- **纠错反馈**：用户可对错误识别结果进行纠正标记，积累数据用于模型微调
- **历史记录**：保存最近识别记录，支持一键复制识别结果
